Подходы к вёрстке под разные устройства
- Резиновая вёрстка -  элементы растягиваются пропорционально, размеры задаются в относительных единицах (% , vw, vh).  
- Адаптивная вёрстка - элементы перестраиваются по правилам (медиа-запросы) при изменении ширины окна.  
- Отзывчивая вёрстка - сочетает оба подхода: плавно подстраивается и перестраивается при нужных точках.  
Без JavaScript сайт реагирует тремя способами:  
1. Относительные единицы измерения (% , vw, vh);  
2. CSS-правила @media;  
3. Показ или скрытие элементов (display: none).

Грид-контейнер (grid container) - элемент с display: grid или inline-grid.
Грид-элементы (grid items) - прямые потомки контейнера.
Грид-линии (grid lines) - невидимые линии, создают сетку; нумеруются с 1.
Грид-строки и колонки (grid tracks) — области между линиями.
Грид-ячейки (grid cells) - пересечение строк и колонок (ячейки сетки).
Grid управляет двумя осями одновременно - строками и колонками,
поэтому подходит для сложных макетов без множества обёрток (в отличие от flex)
grid-template-columns - задаёт ширину столбцов
grid-template-rows - задаёт высоту строк
Размеры можно указывать в px, %, fr, auto
auto - подстраивается под самый большой элемент
В гриде есть две равнозначные оси: строки (вертикаль) и колонки (горизонталь)
Выравнивание всей сетки внутри контейнера:  align-content -  по вертикальной оси (ряды) justify-content -  по горизонтальной оси (колонки)
начения: start, end, center, space-between, space-around, space-evenly
Сокращённая запись: place-content: <align> <justify>; Первое значение - вертикаль, второе - горизонталь. Одно значение применяется к обеим осям
.container {   align-content: center; /* по вертикали */ 
justify-content: space-between; /* по горизонтали */ }
/* то же самое через place-content */   .container {   place-content: center space-between; }

Переполнение блока (overflow)
Если содержимое не помещается в заданные размеры блока, используют свойство `overflow`. 
Значения:
`visible` - по умолчанию, содержимое выходит за границы;  
`hidden` - скрывает лишнее;
`scroll` - всегда показывает полосы прокрутки;
`auto` - добавляет скролл только при переполнении;
`clip` - обрезает содержимое, даже без возможности прокрутки
По осям:
`overflow-x: scroll;   /* горизонтально */ 
overflow-y: hidden;   /* вертикально */ 
overflow: hidden scroll; /* X - скрыт, Y - прокручивается */

Выпадение полей (margin collapse)
Если у родителя нет границ, `padding` или `overflow`, вертикальные отступы дочерних элементов «выпадают» за его пределы.  
Чтобы избежать:
добавить `border: 1px solid transparent;`

Позиционирование элементов 
position: static;     /* по умолчанию: элемент в нормальном потоке, сдвигается только отступами */
Относительное позиционирование
position: relative;   
/* элемент остаётся в потоке, отведённое место сохраняется */
/* смещается относительно своего исходного положения: */
top→ вниз; bottom → вверх; left → вправо; right→ влево
Абсолютное позиционирование 
position: absolute;  /* элемент вырывается из потока, место в потоке остаётся пустым */
/* Позиционируется относительно ближайшего родителя с position ≠ static */
.parent { 
  position: relative; /* родитель с нестатическим позиционированием */
}
.child {
  position: absolute;
  top: 30px;    /* смещает вниз от верхней границы родителя */
  right: 0;     /* смещает от правой границы родителя */
  bottom: 0;    /* смещает вверх от нижней границы родителя */
  left: 40px;   /* смещает вправо от левой границы родителя */
}
/* Если родитель статичен, абсолютный элемент ориентируется на body */
/* z-index — управление глубиной элементов (ось Z) Элементы можно накладывать друг на друга, задавая одинаковые grid-column и grid-row */
position: relative|absolute|fixed|sticky; /* z-index работает только на позиционированных элементах */
z-index: число;   /* больше число → ближе к пользователю, меньше → дальше */ /* по умолчанию auto */
/* Контексты наложения (stacking context) */
- создаются позиционированием, flex или grid
- элементы внутри собственного контекста управляют z-index независимо от внешнего потока
- порядок по умолчанию: элементы ниже в разметке → выше в стопке
Фиксированное позиционирование 
position: fixed;  /* элемент вырывается из потока, поверх других элементов */
/* позиционируется относительно окна браузера, не родителя */
top, right, bottom, left - задают положение /* top и left имеют приоритет над bottom и right */
Липкое позиционирование 
position: sticky;  
/* элемент ведёт себя как static, пока не дойдёт до указанной границы, затем фиксируется */
/* «прилипает» к окну внутри границ родителя */
Свойство inset — шорткат для top, right, bottom, left 
inset: значение;  
/* сокращает запись всех четырёх сторон одновременно */

/* Примеры */
div { inset: 0; }  
/* эквивалентно: top:0; right:0; bottom:0; left:0; */
